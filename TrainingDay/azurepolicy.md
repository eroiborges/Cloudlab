# Laboratorio Azure Day - Azure Policy

## GuardRails

O principal objetivo de utilizar o modelo de politicas é criar os controles de segurança e provisionamento para os diferentes perfis de subscriptions, atuando como um controle adicional ao modelo de permissões RBAC, garantindo que os controles e especificações de provisionamento sejam respeitadas, independente do usuario ter ou não ter permissões para executar uma atividade.

## Azure Policy

Durante o treinamento vamos explorar os diferentes conceitos e cenarios do controle com Azure Policy e também como o Defender for Cloud implementa **Regulatory compliance** através das Azure Policy.

1. No portal do Azure buscar por **Policy**.
2. No menu explorar os itens **Definition** e **Assignements**
3. No menu explorar o item **Remediation**

## Defender for Cloud

1. No portal do Azure buscar por **Microsoft Defender for cloud**
2. No menu explorar o item **Security posture**
3. No menu explorar o item **Environment settings** > no **Management Group** nos 3 pontos clicar em **Edit Settings** e vamos explorar o **Security policies**
